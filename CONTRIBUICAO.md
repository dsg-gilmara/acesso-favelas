# Orientações sobre como contribuir

Contribuições para este código são bem-vindas e apreciadas.
Por favor, siga sempre o nosso [Código de Conduta](./CODIGO_DE_CONDUTA.md).

> Todas as contribuições para este código serão publicadas sob os termos da [LICENÇA](./LICENCA) deste projeto. Ao enviar um *pull request* ou registrar um bug, problema ou solicitação de funcionalidade (*feature request*), você concorda em cumprir esta renúncia de interesse de direitos autorais. Detalhes podem ser encontrados em nossa [LICENÇA](./LICENSE).

Existem duas formas principais de contribuir:

1. Usando o rastreador de *issues*
2. Alterando o código-fonte

## Usando o rastreador de issues

Use o rastreador de *issues* para sugerir novas funcionalidades, relatar bugs e fazer perguntas.
Essa também é uma ótima maneira de se conectar com os desenvolvedores do projeto, bem como com outras pessoas interessadas nesta solução.

Use o rastreador de *issues* para encontrar formas de contribuir.
Se encontrar um bug ou uma funcionalidade, mencione na *issue* que você irá assumir essa tarefa e, em seguida, siga as orientações de *Alterando o código-fonte* abaixo.

## Alterando o código-fonte

De modo geral, você deve fazer um **fork** deste repositório, realizar as alterações no seu próprio *fork* e depois enviar um **pull request**.

Todo novo código deve ter **testes unitários associados** (quando aplicável) que validem as funcionalidades implementadas e a presença ou ausência de defeitos.

Além disso, o código deve seguir quaisquer diretrizes **de estilo e arquitetura** definidas pelo projeto. Na ausência dessas diretrizes, procure **seguir os estilos e padrões já existentes no código do projeto**.
